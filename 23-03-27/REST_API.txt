Q : REST API란 무엇인가요? (모두가 답변) 프로젝트에 REST API를 사용한 이유가 무엇인가요? (프로젝트에 REST API를 사용하셨을 경우에만 추가 답변), REST API 말고 다른 비교할만한 것을 알고 있나요?
A : REST API는 네트워크 상에서 클라이언트와 서버 간에 데이터를 교환하기 위한 인터페이스입니다. 일반적으로 HTTP 프로토콜을 사용하여 통신을 수행하며 리소스 지향 구조(ROA)를 핵심 원칙으로 삼아 각 자원에 고유한 URI를 부여함으로써 클라이언트가 해당 자원에 접근하고
HTTP 메서드를 사용하여 자원을 읽거나 수정할 수 있습니다. 
REST API는 stateless한 특징에 따라 서버의 부하를 줄이고 확장성을 높일 수 있습니다. 트래픽이 증가하거나 시스템의 요구사항이 변경되더라도 쉽게 대응할 수 있는 점, 그리고 HTTP 프로토콜의 인프라를 그대로 사용한다는 점으로 별도의 인프라를 구축할 필요가 없는 이유로
채택하게 되었습니다.
GraphQL을 고려해볼 수 있습니다. REST API의 오버페칭,언더페칭의 문제를 해결할 수 있으며 REST API에 비해 한개의 엔드포인트를 가지고 쿼리에따라 응답이 다르며 원하는 데이터만 가져올 수 있는 장점이 있습니다. 이로써 HTTP 요청 횟수를 줄일 수 있으며 적절한 
HTTP 응답 사이즈를 채택할 수 있습니다. 하지만 캐싱과 파일 업로드 구현 방법이 REST API보다 복잡한 단점이 존재합니다.
